# Plan: Credit System Foundation & Stripe Integration

## Objective

Set up the complete billing infrastructure: database schema for credits and subscriptions, Stripe integration for checkout, webhook handling for subscription lifecycle, and credit consumption enforcement on all AI actions.

## Context

- @src/lib/db/schema.ts — existing schema with users, sessions, accounts, projects, images
- @src/app/actions/ai-image.ts — AI server actions that need credit gating
- @src/lib/auth.ts — Better Auth setup for user identity
- @src/lib/env.ts — environment variable validation

## Tasks

### Task 1: Install Stripe packages and add schema + credit utilities
type="auto"

**Do:**
1. Install `stripe` and `@stripe/stripe-js` packages
2. Add STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET, NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY to env.ts
3. Create `src/lib/stripe.ts` with Stripe server client
4. Add to `src/lib/db/schema.ts`:
   - `plans` table: id, name, stripePriceId, credits, price, interval
   - `subscriptions` table: id, userId, planId, stripeCustomerId, stripeSubscriptionId, status, currentPeriodEnd, createdAt, updatedAt
   - `creditBalances` table: id, userId, balance, updatedAt
   - `creditTransactions` table: id, userId, amount, type (grant/deduct/purchase), description, createdAt
5. Create `src/lib/credits.ts` with:
   - `CREDIT_COSTS` map: removeBackground=2, removeObject=3, generateBackground=3, detectText=1, translateText=1
   - `getCreditBalance(userId)` — query current balance
   - `checkAndDeductCredits(userId, cost)` — atomic check + deduct in transaction, throws if insufficient
   - `addCredits(userId, amount, type, description)` — add credits and log transaction

**Verify:** TypeScript compiles, schema types are correct

**Done when:** Schema tables defined, credit utilities export correctly, stripe client initializes

### Task 2: Stripe webhook handler and billing server actions
type="auto"

**Do:**
1. Create `src/app/api/stripe/webhook/route.ts`:
   - Verify Stripe signature
   - Handle `checkout.session.completed`: create subscription record, grant initial credits
   - Handle `invoice.paid`: grant monthly credits for recurring subscriptions
   - Handle `customer.subscription.updated`: update subscription status
   - Handle `customer.subscription.deleted`: mark subscription cancelled
2. Create `src/app/actions/billing.ts`:
   - `createCheckoutSession(planId)`: create Stripe Checkout session for subscription
   - `purchaseCredits(packSize)`: create Stripe Checkout session for one-time credit pack (10 credits=$5, 50=$20, 100=$35)
   - `getSubscriptionStatus()`: return current user's subscription and credit balance
   - `createCustomerPortalSession()`: return Stripe Customer Portal URL for managing subscription
3. Update `src/app/actions/ai-image.ts`:
   - Add `checkAndDeductCredits` call at the start of each AI action (after auth check, before AI API call)
   - Each action deducts its cost from CREDIT_COSTS

**Verify:** TypeScript compiles, webhook route exports POST handler, billing actions export correctly

**Done when:** Webhook processes Stripe events, billing actions create checkout sessions, AI actions consume credits

## Verification

- All files compile without type errors
- Schema has all required tables
- Credit deduction is called in every AI action
- Webhook handles all subscription lifecycle events

## Success Criteria

1. Database schema includes subscriptions, credit_balances, credit_transactions tables
2. Stripe webhook handles checkout.session.completed, invoice.paid, subscription.updated/deleted
3. Every AI action checks and deducts credits before calling the AI API
4. Server actions exist for creating checkout sessions and purchasing credits
