  <action>
Create `src/lib/credits.ts` with `"use server"` directive per D-17.

1. Define and export `CREDIT_COSTS` constant per D-10:
   ```typescript
   export const CREDIT_COSTS = {
     "background-removal": 1,
     "object-removal": 2,
     "ai-background": 2,
     "text-detection": 1,
     "text-replacement": 3,
     "translation": 1,
     "upscale": 2,
     "style-transfer": 2,
   } as const;
   export type CreditOperation = keyof typeof CREDIT_COSTS;
   ```

2. Export `PLAN_CREDITS` constant per D-06:
   ```typescript
   export const PLAN_CREDITS = { free: 50, pro: 500, enterprise: Infinity } as const;
   ```

3. Implement `getCreditBalance(userId: string)`:
   - Query creditBalances where userId matches
   - If no row exists (handles users who signed up before credit system, per Pitfall 6 in RESEARCH), create one with `monthlyCredits: 50`, `bonusCredits: 0`, `resetAt: addMonths(new Date(), 1)` using upsert pattern
   - Return `{ monthlyCredits, bonusCredits, total: monthlyCredits + bonusCredits, plan }`
   - Also join/query the subscriptions table to include the user's plan name
   - Import `addMonths` from `date-fns`

4. Implement `checkCredits(userId: string, operation: CreditOperation)` per D-11, D-29:
   - Call getCreditBalance to get current balance
   - Look up cost from CREDIT_COSTS
   - If `plan === "enterprise"` return `{ cost: 0, available: Infinity }` (unlimited)
   - If `total < cost`, throw error `"Insufficient credits: need ${cost}, have ${total}"`
   - Return `{ cost, available: total }`

5. Implement `deductCredits(userId: string, operation: CreditOperation, cost: number)` per D-12, D-30:
   - Use `db.transaction()` for atomicity (Pitfall 2 in RESEARCH)
   - Inside transaction: read balance with fresh query
   - Deduct from monthlyCredits first, then bonusCredits for remainder per D-14
   - `const monthlyDeduct = Math.min(balance.monthlyCredits, cost)`
   - `const bonusDeduct = cost - monthlyDeduct`
   - Update creditBalances with decremented values using `sql` template
   - Insert creditTransactions row with type "deduction", amount -cost, operation name
   - Use `eq` and `sql` from drizzle-orm

6. Implement `getUsageHistory(userId: string, days = 30)`:
   - Query creditTransactions for given userId ordered by createdAt desc
   - Filter to last N days using `gte(creditTransactions.createdAt, subDays(new Date(), days))`
   - Return array of `{ id, type, amount, operation, createdAt }`
   - Import `subDays` from `date-fns`
  </action>
