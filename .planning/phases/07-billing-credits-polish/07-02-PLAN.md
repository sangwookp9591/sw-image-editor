Create `src/app/api/stripe/checkout/route.ts` with a POST handler per D-01.

1. Authenticate user via `auth.api.getSession({ headers: await headers() })`. Return 401 if no session.

2. Parse request body: `{ priceId: string, mode: "subscription" | "payment" }`.
   - `mode: "subscription"` for Pro/Enterprise plan signup per D-09
   - `mode: "payment"` for one-time credit top-up per D-13

3. Check if user already has a Stripe customer ID by querying the `subscriptions` table:
   - If exists, use that `stripeCustomerId`
   - If not, create a new Stripe customer via `stripe.customers.create({ email: session.user.email, metadata: { userId: session.user.id } })`
   - Upsert a subscriptions row with the new stripeCustomerId (plan: "free", status: "active")

4. Create Checkout session via `stripe.checkout.sessions.create()`:
   ```typescript
   {
     mode: mode,
     customer: stripeCustomerId,
     line_items: [{ price: priceId, quantity: 1 }],
     success_url: `${process.env.NEXT_PUBLIC_APP_URL}/dashboard/usage?success=true`,
     cancel_url: `${process.env.NEXT_PUBLIC_APP_URL}/dashboard/usage?canceled=true`,
     metadata: { userId: session.user.id },
   }
   ```
   - For subscription mode, add `subscription_data: { metadata: { userId: session.user.id } }`

5. Return `NextResponse.json({ url: checkoutSession.url })`.

6. Add Billing Portal session creation as a second endpoint. Add a second exported handler or handle via a query param. Actually, create a separate function `createPortalSession`:
   - Add a GET handler that creates a `stripe.billingPortal.sessions.create({ customer: stripeCustomerId, return_url: ... })` per Open Question 2 in RESEARCH
   - Return the portal URL for redirect

  <action>
Create `src/app/api/stripe/webhook/route.ts` per D-02, D-03.

1. Parse raw body with `req.text()` (NOT `req.json()` -- Pitfall 1 in RESEARCH).

2. Verify signature with `stripe.webhooks.constructEvent(body, signature, process.env.STRIPE_WEBHOOK_SECRET!)`. Return 400 on failure.

3. Handle events per D-03:

   **`checkout.session.completed`:**
   - Extract userId from `event.data.object.metadata.userId`
   - If `session.mode === "subscription"`:
     - Get subscription details via `stripe.subscriptions.retrieve(session.subscription)`
     - Determine plan from price ID (match against known Pro/Enterprise price IDs stored as env vars or hardcoded constants -- use constants in this file: `const PRICE_TO_PLAN: Record<string, "pro" | "enterprise"> = {}` and note that actual price IDs must be filled in after Stripe product setup)
     - Upsert subscriptions table: set stripeSubscriptionId, plan, status, currentPeriodEnd
     - Reset creditBalances.monthlyCredits to PLAN_CREDITS[plan] value
     - Log creditTransactions with type "plan_change"
   - If `session.mode === "payment"` (top-up per D-13):
     - Extract credit amount from session metadata or line item (store credit amount in Checkout metadata when creating session)
     - Add to creditBalances.bonusCredits (these don't expire per D-14)
     - Log creditTransactions with type "top_up"

   **`invoice.paid`:**
   - Extract subscription ID from invoice
   - Look up user via subscriptions.stripeSubscriptionId
   - Reset monthlyCredits to PLAN_CREDITS[plan] per D-07
   - Update creditBalances.resetAt to next billing cycle
   - Log creditTransactions with type "monthly_reset"
   - Use idempotent SET (not increment) to prevent double-credit on webhook retry (Pitfall 3)

   **`customer.subscription.updated`:**
   - Extract subscription from event
   - Update subscriptions table: plan, status, currentPeriodEnd
   - If plan changed, adjust monthlyCredits to new PLAN_CREDITS[plan]

   **`customer.subscription.deleted`:**
   - Find subscription by stripeSubscriptionId
   - Update plan to "free", status to "canceled"
   - Set monthlyCredits to 50 (free tier per D-06, D-08)
   - Log creditTransactions with type "plan_change"

4. Return `new Response("OK", { status: 200 })` for all events (including unhandled ones).

5. Export the route config to disable body parsing if needed. In Next.js App Router route handlers, `req.text()` works without config changes.
