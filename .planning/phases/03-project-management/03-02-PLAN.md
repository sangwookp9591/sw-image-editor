2. Create `src/components/editor/hooks/use-save.ts` — a custom hook that orchestrates the full save flow:

   ```typescript
   export function useSave(fabricRef: RefObject<FabricCanvas | null>)
   ```

   Returns: `{ save: (nameOverride?: string) => Promise<void>, saveAs: () => void, isSaving: boolean }`

   **save(nameOverride?)** implementation:
   a. Get current state from useEditorStore: projectId, projectName
   b. If projectId is null AND no nameOverride provided, this is a first save — set a flag to show the save dialog (store a `pendingSave` state or return early, letting the dialog handle it). Use a pattern like: if no projectId and no nameOverride, call a `onNeedName` callback (passed as parameter or use a ref).

   SIMPLER APPROACH: The hook returns `{ save, saveAs, needsName }` where `needsName` is a boolean state. When save() is called and projectId is null, set needsName=true and return. The dialog reads needsName, collects the name, then calls save(name). When save(name) is called with a name AND projectId is null, proceed with create.

   c. Set saveStatus to "saving"
   d. Serialize canvas: `const canvasJson = JSON.stringify(fabricRef.current!.toJSON())`
   e. Generate thumbnail (non-blocking — save succeeds even if thumbnail fails):
      - `const dataUrl = fabricRef.current!.toDataURL({ format: "jpeg", quality: 0.6, multiplier: 200 / Math.max(fabricRef.current!.getWidth(), fabricRef.current!.getHeight()) })`
      - Convert to blob: `const res = await fetch(dataUrl); const blob = await res.blob();`
      - Get presigned URL: `const { presignedUrl, key } = await fetch("/api/upload/thumbnail", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ contentType: "image/jpeg" }) }).then(r => r.json())`
      - Upload: `await fetch(presignedUrl, { method: "PUT", body: blob, headers: { "Content-Type": "image/jpeg" } })`
      - Wrap entire thumbnail block in try/catch — on failure, set thumbnailKey to null
   f. Call saveProject server action: `const result = await saveProject({ projectId, name: nameOverride ?? projectName ?? "Untitled", canvasJson, thumbnailKey })`
   g. Update store: setProjectId(result.projectId), setProjectName(name), setSaveStatus("saved")
   h. After 2 seconds, set saveStatus back to "idle" (debounce with setTimeout)
   i. On error: setSaveStatus("error"), show toast via sonner if available (check if sonner is installed first — if not, just console.error)

   **saveAs()**: Set needsName=true regardless of current projectId. When dialog confirms, create a NEW project (call save with projectId forced to null). After save, update URL to new project route via `router.push(/editor/project/${newId})`.

   Add `isSaving` guard: if saveStatus is "saving", return early from save() to prevent double-save (per Pitfall 7).

   Import `useRouter` from `next/navigation` for saveAs redirect.
