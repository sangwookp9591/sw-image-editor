1. Create `src/app/actions/projects.ts` with `"use server"` directive containing two Server Actions:

   **saveProject:**
   ```
   export async function saveProject(input: {
     projectId: string | null;
     name: string;
     canvasJson: string;
     thumbnailKey: string | null;
   }): Promise<{ projectId: string }>
   ```
   - Auth check: `const session = await auth.api.getSession({ headers: await headers() })` — throw if no session
   - If `input.projectId` is non-null (subsequent save per D-03): UPDATE projects SET name, canvasJson, thumbnailKey, updatedAt=new Date() WHERE id=projectId AND userId=session.user.id
   - If `input.projectId` is null (first save per D-02): INSERT into projects with values {userId, name, canvasJson, thumbnailKey}, use `.returning()` to get the new id
   - Call `revalidatePath("/")` after mutation to refresh dashboard
   - Return `{ projectId }` (either existing or newly created)

   **deleteProject:**
   ```
   export async function deleteProject(projectId: string): Promise<void>
   ```
   - Auth check same pattern
   - Fetch project first to verify ownership: `db.select().from(projects).where(and(eq(projects.id, projectId), eq(projects.userId, session.user.id)))`
   - If not found, throw error
   - If project.thumbnailKey exists, delete from S3: `s3Client.send(new DeleteObjectCommand({ Bucket: BUCKET, Key: project.thumbnailKey }))` (per D-16)
   - Delete DB record: `db.delete(projects).where(eq(projects.id, projectId))` — this cascades to images via FK (per D-16)
   - Call `revalidatePath("/")`

   Import DeleteObjectCommand from `@aws-sdk/client-s3`. Import s3Client and BUCKET from `@/lib/s3`.

2. Create `src/app/api/upload/thumbnail/route.ts` — POST route handler:
   - Auth check (same pattern)
   - Generate S3 key: `thumbnails/${crypto.randomUUID()}.jpg`
   - Create presigned PUT URL using the existing `createPresignedUploadUrl` helper from `src/lib/s3.ts`, passing contentType "image/jpeg" and filename like "thumbnail.jpg"
   - Return JSON: `{ presignedUrl, key }`
   - NOTE: The existing createPresignedUploadUrl uses the fileName extension to build the key with "uploads/" prefix. For thumbnails, either:
     (a) Create a new helper `createThumbnailPresignedUrl` that uses "thumbnails/" prefix, OR
     (b) Inline the PutObjectCommand + getSignedUrl directly in the route with "thumbnails/" prefix
   - Option (b) is simpler for a single use case. Use it.
